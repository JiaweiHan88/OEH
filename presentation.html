open_engineering_hub // 01

Knowledge evaporates. Every day. At scale.

Engineering teams in large enterprises suffer a compounding fragmentation problem existing tools don't solve.

// problem_01

Disconnected systems

Jira, Confluence, GitHub, Gerrit, Codebeamer, Zuul. Context lives everywhere and nowhere.

// problem_02

Up to 40% of time lost to context assembly

Engineers juggle dozens of tools and browser tabs to understand a single defect, requirement, or change.

// problem_03

Zero compounding

A senior finds a root cause, closes the ticket, the insight evaporates. Next engineer starts from zero.

// problem_04

Knowledge walks out the door

Attrition, reorgs, and handoffs erase months of accumulated understanding.

This isn't a tab-switching problem. It isn't a chatbot problem. It's a knowledge architecture problem — and no existing tool solves it.

6_layer // 04

Breakthrough #1 — multi-strategy retrieval

A single retrieval strategy always fails at something. OEH runs three — in parallel — and fuses the results.

// [ keyword ]

PostgreSQL BM25

Error codes, identifiers, exact terms.

> find error 0xDEAD in all Jira tickets

// [ semantic ]

pgvector embeddings

Conceptual similarity, natural language.

> issues where display flickers on cold boot

// [ graph ]

Property graph

Relationships, traceability, multi-hop.

> what requirements trace to this failing test?

   query ──► BM25  ──┐
             pgvec ──┼──► Reciprocal Rank Fusion ──► merged results
             graph ──┘

retrieval // 05

Breakthrough #2 — agents that learn & remember

The agents are not stateless chatbots. They learn. They remember. They get smarter.

// L3a :: agent memory

Topic-indexed knowledge base

defect_analysis/ (user: lena)
├── _index.md              ← always in ctx
├── vhal-binder-patterns.md
├── dlt-log-parsing-lessons.md
└── common-aaos-failure-modes.md
  • Always-loaded index — compact overview every turn
  • On-demand topics — loaded when task matches
  • Self-managed — agent writes, merges, archives
  • Deterministic — no lossy LLM summarization; [CONFLICT] tags

// L3b :: cross-session recall

Full-text search across past conversations

"Last time we looked at VHAL issues, root cause was always in the binder transaction layer."

// L3c :: procedural skills

Reusable workflows from experience

skill: DLT-VHAL-Triage
trigger: VHAL defect + DLT log
steps:
  01. parse DLT for VHAL PIDs
  02. filter err-level in 5s window
  03. x-ref known failure modes
  04. check Codebeamer reqs
  05. emit root-cause report

// After 3 months of use, the agent has seen 200+ triage sessions. New team members get the benefit on day one.

self_learning // 06

Breakthrough #3 — the Hive

Individual learning is powerful. Shared learning is transformative. The Hive turns individual agent skills into organizational knowledge via a community PR model.

DRAFT

agent discovers

──▸

PRIVATE

L3c skill

──▸

PROPOSED

engineer submits

──▸

APPROVED

privacy + review

──▸

PUBLISHED

all agents gain it

// gate_01

Privacy scanning

Regex-based credential & PII detection before publish.

// gate_02

Applicability tags

Tech stack, domain, project — agents load only relevant skills.

// gate_03

Relevance scoring

Workspace context matched against tags; usage feedback ranks skills.

// gate_04

Provenance

Every shared skill traces back to origin & author.

// 12 engineers × daily use = dozens of verified patterns/month flowing into the Hive. Senior expertise scales without senior interrupts.
